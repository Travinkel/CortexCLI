W40: Session
About this week
We can authenticate user. Now what?

HTTP is a stateless protocol. So, how do we keep track of users across requests? We don't want our users to have to re-authentication each time they click something, right?

The answer is session management.

We will discuss tokens and cookies.

W42: Autumn holiday
W43: Web security
About this week
Welcome back from vacation. Hope you enjoyed it.

We are just going to jump right into a deeply technical topic. This week is all about "hacking". Yes you read correctly, that is exactly what we will be doing do. You will all be competing in a hacking competition. Better do your homework. Remember "Learning resources" means homework that you need to do before class.

The idea is that by discussing web security from an attackers point of way, you will learn to think like an attacker, thereby teaching you the foundation for what to protect against.

We will also touch on what we can do as a developer to protect against these attacks.

Don't try any of this on other peoples systems. It is illegal and will get you in jail!
